feat(seo): 優化 Lighthouse 報告中的 SEO 及最佳實踐問題

本次提交解決了 Lighthouse 報告中提出的多個問題，以提升網站的 SEO 表現和最佳實踐分數：

- **Meta Description 優化**：
  - 修正 `app/(public)/blog/[id]/page.tsx` 中動態生成文章 Meta Description 的邏輯，確保其內容更完整。
  - 在動態生成的 Meta Description 過短時，使用 `SITE_CONFIG.description` 作為備用，避免空白描述。

- **Canonical URL 優化**：
  - 從 `app/layout.tsx` 中移除全域的 `alternates.canonical: '/'` 設定，確保每個頁面都能正確設定其標準連結，避免所有頁面都指向首頁。

- **Hydration Mismatch 修復**：
  - 透過在 `app/providers.tsx` 中動態導入 `sonner` 的 `Toaster` 組件並設定 `ssr: false`，解決了由於 `sonner` 在伺服器端渲染時注入樣式導致的水合不匹配問題。

- **錯誤頁面處理**：
  - 新增 `app/error.tsx` 和 `app/not-found.tsx`，以符合 Next.js App Router 的錯誤處理機制，解決了 `_error` 頁面找不到的建置錯誤。

這些改動有助於提升網站的搜尋引擎可見性、使用者體驗和程式碼健壯性。
